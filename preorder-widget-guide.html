Requirements

Before You Start

Three things must be in place or the widget won't show up at all.

RequirementWhere to check
Essential Preorder app installed Shopify Admin → Apps
App embed turned ON Theme Editor → App embeds panel → toggle on
Discount configured in the app Essential Preorder App → Content →Discount
No compare-at price = widget stays hidden. The widget only shows when there is an actual discount to display. Set a compare-at price on every preorder variant.
Installation

Two Files, Four Steps

  1. Upload the snippet

    In Shopify Admin go to Online Store → Themes → Edit code. Inside the Snippets folder, create a new file:

    preorder-discount-widget.liquid

    Paste the full snippet found here : content and save.

  2. Open buy-buttons.liquid

    Find this file in your theme. For Horizon it lives at:

    blocks/buy-buttons.liquid

    For other themes see the theme table at the bottom of this guide.

  3. Add the render tag — above the Add to Cart button

    Find the product-form-buttons div (or the add-to-cart button block). Place the render tag directly above it, still inside the form:

    {%- comment -%} Preorder Discount Widget {%- endcomment -%}
{% render 'preorder-discount-widget' %}

<!-- add-to-cart button comes right after this -->
<div class="product-form-buttons">
  ...
</div>
    Must be inside the form block. The tag must sit between {%- form -%} and {%- endform -%}, or variant switching won't update the widget.
  4. Enable the app embed & set discount in the app

    In Theme Editor open App embeds and toggle on Essential Preorder & Back In Stock. Then in the product editor, Then in the Essential Preorder App, set a discount.

    Done. Visit and reload the product page. The widget appears automatically above the Add to Cart button whenever a preorder variant with a discount is selected.
Optional

Change the Discount Format

At the very top of the snippet there is one variable that controls how the badge reads:

{% comment %} 'percent' → Save 38%  |  'amount' → Save $30.00 {% endcomment %}
{% assign pw_discount_format = 'percent' %}

Change 'percent' to 'amount' to show the flat currency saving instead. This is for merchants who choose to use amount off ' instead of ' percent off in the discount. The currency symbol is pulled automatically

Logic

When the Widget Shows vs Hides

The widget checks two things every time a variant is selected:

1. Is this variant in preorder state?
   → Looks for .essential-preorder-container-active in the DOM

2. Is there a real discount?
   → compare-at price exists AND is higher than preorder price

Both true  →  widget shows
Either false →  widget hides (no empty space left behind)

The widget updates automatically when the customer switches variants. No page reload needed.

Troubleshooting

Common Issues

Widget not showing at all
App embed is off, or no compare-at price set on the variant
Strikethrough price missing
Discount within the app is not set
Widget shows on non-preorder variants
App isn't marking the variant as inactive, check if essential-preorder-container-active in the DOM, but not displaying the preorder button
Doesn't update when switching variants
Render tag is outside the {%- form -%} block — move it inside
Liquid error: snippet not found
File must be named exactly preorder-discount-widget.liquid (all lowercase, hyphens)
Other Themes

Where to Put the Render Tag by Theme

The rule is always the same: inside the product form, above the add-to-cart button.

ThemeFile to editPlace render tag before
Horizon blocks/buy-buttons.liquid product-form-buttons div
Dawn snippets/buy-buttons.liquid Add to cart button block
Impulse snippets/product-form.liquid Add to cart button
Prestige sections/product.liquid buy_buttons block render
Debut sections/product-template.liquid form "product" open tag
Any other theme Search codebase for type="add-to-cart-form" Above the add-to-cart button, inside the form
Note: At the end of the day, it will always depends on merchants theme’s structure and where the Add to Cart (ATC) button is placed. The preorder button is tied to that same element, so its placement follows wherever the ATC button is defined in your theme.
Testing

Quick Checklist Before Going Live

  • Widget appears above Add to Cart on a preorder variant
  • Left card shows the early-bird price with discount already (no strikethrough)
  • Right card shows the original price with strikethrough
  • Badge shows the correct % or $ saving
  • Widget hides when a non-preorder variant is selected
  • Widget reappears when switching back to preorder variant with the discount set
  • On mobile — both cards side by side, text readable
